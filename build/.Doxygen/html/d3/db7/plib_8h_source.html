415/* to understand why this macro exists along with its implicit
416 * pl_arg_global_ptr. originally pl_a returned a pointer pl_arg
417 * to the actual item in the arguments array, this was very bad
418 * as when the argument list reached capacity it would overwrite
419 * the pointers memory location which creates a dangling pointer.
420 * this pointer can still be interacted with without error, the
421 * issue with this is that the pointer that we think is pointing
422 * to an argument in our argument list is actually not existant
423 * which means that from a debug point of view if you created an
424 * argument and the list was re-allocated then now for some
425 * reason regardless of how you trigger that argument it still
426 * comes up as false when using pl_arg_run this is of course
427 * because the pointer is refering to something other then what
428 * it is suppost too. to combat this i changed the code so an
429 * index is returned instead of a pointer, then after all of the
430 * args are set you can use either &PL_ARGS[index] or PL_GETI(i)
431 * to grab a pointer to an argument, now this was a good fix but
432 * now one line has turned into two lines, one for definition
433 * and a second for actually getting the pointer from the index.
434 * the way this is resolved is by using the following macro which
435 * is a copy of the PL_A macro but for the pl_arg_global_ptr func
436 * the pl_arg_global_ptr function will first check to see if the
437 * arg has been created if it has then it will search for it in
438 * the argument list and return a pointer to it if it finds it.
439 * if the arg has not been made then the arg will be offloaded
440 * into the pl_a function and then returned as a pointer. this
441 * means that if you initialize a var using the PL_P function
442 * every time you call the variable it will perform a search in
443 * the argument list for the argument which removes all risk of
444 * dangling pointers as the information is updated per-call and
445 * isent static and overwritable. */
446#define PL_P(...) pl_arg_global_ptr((pl_arg){__VA_ARGS__})
447pl_arg *pl_arg_global_ptr(pl_arg in);

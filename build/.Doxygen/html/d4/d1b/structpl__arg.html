argument options

Parameters
nameflag name eg --help
descriptionflag description (used in pl_help)
takes_valueset whether or not the argument should expect a value
catagoryset a catagory for an argument, arguments of the same catagory are printed togeather in pl_help
typetype of value, not actuall enforced but used in pl_help instead. eg VOID or INT.
valueif .takes_value = 1 then the value(s) parsed in will be allocated here.
value_capValue is dynamically allocated this is a variable used in the DMA.
value_idxsame as value_cap but this value also is the amount of arguments parsed into the flag, this doubles as the run count of any arguments that take a value.
triggeredthis value is >1 if the argument was run and is 0 if the arg was not run. this doubles as the run count for all arguments

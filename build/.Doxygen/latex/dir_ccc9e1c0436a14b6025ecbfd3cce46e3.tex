\hypertarget{README.md_autotoc_md4}{}\doxysubsubsection{\texorpdfstring{Features}{Features}}\label{README.md_autotoc_md4}
Even though both snippets are roughly the same size the plib option offers much more readable code as well as having indepth builtin error detection. plib comes out of the box with a custom help function that prints out all of the arguments in a nice format like so\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{Options:\ }
\DoxyCodeLine{\ \ -\/-\/help\ |\ NO\ VALUE}
\DoxyCodeLine{\ \ -\/-\/test\ |\ VALUE\ }

\end{DoxyCode}


This help menu prints out all information available for an argument, other options that can be added to an argument include,
\begin{DoxyItemize}
\item Catagory\+: \begin{quote}
defaults to Options\+: and is used as the argument header. \end{quote}

\item Type\+: \begin{quote}
type of argument, used only in the plib help dialog \end{quote}

\item Description\+: \begin{quote}
description of argument \end{quote}

\item Takes Value\+: \begin{quote}
if the argument takes a value or not. \end{quote}

\end{DoxyItemize}
